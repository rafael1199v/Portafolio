What is GraphQL?

GraphQL is a query language developed by Facebook in 2012 that makes data fetching more efficient to meet the needs of modern web applications. It allows clients to request exactly the data they need, which can reduce the amount of data transferred between the client and server. The schema is the most important part of a GraphQL endpoint. Designing a GraphQL schema is a foundational step in building a GraphQL API. It serves as a contract between the client and the server, detailing the types of data available and the ways clients can interact with that data.

What is a microservice architecture?

A microservice architecture is a flexible model for the design of applications. It works like a group of small services that each complete a unique business task. These microservices can be developed in any programming language and communicate with each other through APIs. This architecture offers advantages in versatility, scalability and code modularity. On the other hand, it can be a significant investment of time at the beginning of a project.

Hexagonal Architecture

Hexagonal Architecture is an architectural pattern that allows external entities or systems to interact with the application through a port via an adapter. This architecture protects the core of the application from external components and aims to be independent of tools and technologies. Ports act as entry points defined by the application that sets the rules for communication with external actors. Anyone who wants to interact with the application should implement that interface. Adapters initiate the interaction with the application. There can be any number of adapters for a port because all implement the same rules.
